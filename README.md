# 实验室管理平台

一个面向课题组/实验室局域网使用的轻量级资料管理平台。当前版本是可运行 MVP，重点覆盖资料上传下载、目录分类、登录鉴权、管理员审核、公告发布、账号管理和多人协作笔记。

## 已实现功能

### 账号与权限

- 支持登录，不支持公开注册。
- 支持保持登录 30 天；未手动退出时，服务重启后仍保持登录。
- 登录页会记住上次使用的用户名，但不会保存明文口令。
- 首次启动自动创建管理员账号。
- 只有管理员可以添加账号、创建目录、审核文件、发布公告。
- 普通用户可以上传文件、查看已发布文件、下载文件、写协作笔记。

### 资料库

- 左侧目录树展示资料分类。
- 管理员可新建、重命名、删除空目录。
- 目录下展示文件列表，点击文件可查看详情。
- 记录文件上传人、上传时间、大小、类型、状态、审核人、审核时间。
- 支持文件下载。
- 支持 PDF 在线预览。
- 支持 Markdown、TXT 在线预览。
- 支持图片预览。

### 上传审核

- 管理员上传：自动进入“已发布”。
- 普通用户上传：自动进入“待审核”。
- 管理员可在管理后台查看待审核文件，并执行通过/拒绝。
- 普通用户可以看到自己上传但未审核的文件状态。

### 公告

- 管理员可发布公告。
- 支持置顶公告。
- 所有登录用户可查看公告。

### 协作笔记

- 支持创建多人笔记。
- 笔记内容使用 Markdown 文本。
- 多人打开同一篇笔记时，通过 Socket.IO 实时广播编辑内容。
- 当前版本采用“实时广播 + 手动保存”的轻量协作模式；如果后续需要严格无冲突协作，可升级为 Yjs/CRDT。

## 技术栈

- 后端：Node.js + Express
- 数据库：Node.js 内置 `node:sqlite`
- 登录会话：express-session + 自定义 SQLite Session Store
- 文件上传：multer 2.x
- 实时协作：Socket.IO
- Markdown 渲染：markdown-it + sanitize-html
- 前端：EJS + 原生 JavaScript + CSS
- 文件存储：服务器本地磁盘 `storage/uploads`

选择这套技术栈的原因：部署简单、维护成本低、不依赖公网服务，适合实验室一台服务器在局域网长期运行。

## 启动方式

进入项目目录：

```bash
cd 实验室管理平台
npm install
npm start
```

默认监听：

```text
http://0.0.0.0:3000
```

在服务器本机访问：

```text
http://localhost:3000
```

在局域网其他电脑访问时，把地址中的 `localhost` 换成服务器内网 IP，例如：

```text
http://192.168.1.10:3000
```

## 环境配置

复制示例配置：

```bash
cp env.example .env
```

建议至少修改：

```text
SESSION_SECRET=自己生成的一段随机字符串
ADMIN_USER=admin
ADMIN_PASSWORD=管理员初始口令
MAX_UPLOAD_MB=200
```

> 注意：如果不配置 `ADMIN_PASSWORD`，系统会使用内置演示口令创建初始管理员。正式使用前应设置 `.env`，并在首次登录后立即修改管理员口令。

## 后续建议增强

### 短期增强

- 文件全文搜索。
- 文件版本管理。
- 文件移动/重命名/删除。
- 目录排序。
- 管理员查看审计日志。
- 支持批量上传。
- 支持个人修改口令的前端入口。

### 中期增强

- 引入 Yjs，实现真正多人无冲突协作编辑。
- 支持 LDAP/学校统一认证。
- 增加设备预约、耗材库存、仪器台账模块。
- 支持按课题/项目建立资料空间。
- 支持定期备份数据库和上传文件。

### 部署建议

- 推荐使用 PM2 或 systemd 保持服务常驻。
- Windows 服务器可用 NSSM 注册为服务。
- 正式使用时建议放到反向代理后面，例如 Nginx/Caddy。
- 即使只在局域网使用，也建议定期备份 `data` 和 `storage` 两个目录。
