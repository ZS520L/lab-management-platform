# 功能设计与技术栈

## 1. 项目定位

本平台定位为“实验室内部资料与协作管理平台”，优先满足局域网部署、账号受控、资料分类、上传审核、下载分发、公告发布和协作笔记需求。

平台不做公开注册，不面向公网开放，适合部署在实验室服务器、办公室电脑或学院内网服务器上。

## 2. 用户角色

### 管理员

- 添加、停用、管理账号。
- 创建资料目录。
- 上传文件并直接发布。
- 审核普通用户上传的文件。
- 发布公告。
- 查看所有目录下的待审核/已拒绝文件。
- 参与协作笔记。

### 普通用户

- 登录平台。
- 浏览已发布资料。
- 下载资料。
- 上传文件到指定目录，但上传后进入待审核状态。
- 查看自己上传文件的审核状态。
- 创建、编辑、保存协作笔记。

## 3. 当前已落地功能

### 3.1 登录与账号管理

- 支持用户名和口令登录。
- 不开放注册入口。
- 首次启动自动初始化管理员账号。
- 管理员可添加普通用户或管理员。
- 账号口令使用 bcrypt 哈希保存。
- 登录会话保存在 SQLite 中。

### 3.2 资料目录树

- 左侧显示树形目录。
- 支持多级目录。
- 管理员可在当前目录下新建子目录。
- 目录显示已发布文件数量；管理员还能看到待审核数量。

### 3.3 文件上传与审核

- 支持上传任意类型文件。
- 管理员上传文件后自动发布。
- 普通用户上传文件后进入待审核区。
- 管理员可通过或拒绝文件。
- 拒绝时记录拒绝原因。
- 文件元数据记录到数据库，真实文件保存到服务器磁盘。

### 3.4 文件详情与预览

- 点击文件显示：文件名、目录、上传人、上传时间、大小、类型、状态、审核信息。
- PDF 使用浏览器内嵌预览。
- Markdown 文件渲染为 HTML 预览。
- TXT 文件以纯文本预览。
- 图片文件直接预览。
- 其他文件支持下载。

### 3.5 公告

- 管理员可发布公告。
- 公告支持置顶。
- 所有登录用户可查看。

### 3.6 协作笔记

- 支持创建笔记。
- 支持 Markdown 文本编辑。
- 多人打开同一笔记时，通过 Socket.IO 广播编辑内容。
- 保存后写入 SQLite。

当前协作模式适合轻量共同编辑。若多人高频同时编辑，需要进一步升级到 Yjs/CRDT，以避免覆盖冲突。

## 4. 建议继续完善的功能

### 第一阶段：资料库增强

- 文件搜索：按文件名、说明、上传人、目录搜索。
- 文件移动：管理员可移动文件到其他目录。
- 文件重命名：管理员可修改文件显示名。
- 文件删除：软删除到回收站，避免误删。
- 文件版本：同名文件可形成版本历史。
- 批量上传：一次上传多个文件。
- 目录排序：管理员可调整目录顺序。

### 第二阶段：管理能力增强

- 审计日志页面：查看登录、上传、下载、审核、公告发布记录。
- 个人中心：用户修改自己的口令。
- 权限细分：按目录设置“可见、可上传、可审核”。
- 课题空间：按课题/项目隔离资料。
- 备份入口：管理员一键导出数据库和上传文件索引。

### 第三阶段：实验室管理模块

- 设备预约：仪器、工位、GPU 服务器预约。
- 耗材库存：试剂、元器件、耗材出入库。
- 设备台账：仪器编号、负责人、维修记录、说明书。
- 入组流程：新成员入组清单、账号申请、培训记录。
- 组会记录：组会纪要、PPT、任务分配。

## 5. 推荐技术栈

### 当前 MVP 技术栈

| 层级 | 技术 | 原因 |
|---|---|---|
| 后端 | Node.js + Express | 开发快、依赖少、适合轻量 Web/API 服务 |
| 数据库 | Node.js 内置 `node:sqlite` | 零额外原生依赖、单文件、便于备份，适合几十人实验室内网系统 |
| 文件存储 | 本地磁盘 | 简单可靠，方便挂载大容量硬盘或 NAS |
| 会话 | express-session + 自定义 SQLite Session Store | 登录状态持久化，不需要额外 Redis 或原生 sqlite3 依赖 |
| 上传 | multer 2.x | Express 生态常用上传中间件 |
| 实时协作 | Socket.IO | 支持 WebSocket 与降级传输，适合局域网实时协作 |
| 模板 | EJS | 无需前端构建流程，部署简单 |
| 前端 | 原生 JS + CSS | 减少构建和维护成本 |
| Markdown | markdown-it + sanitize-html | 支持 Markdown 预览，同时做 HTML 清理 |

### 为什么不一开始上 Vue/React + MySQL/PostgreSQL

如果只是课题组内部使用，系统的第一目标是“稳定可用、容易部署、容易维护”。Vue/React + 独立数据库当然更适合大型系统，但也会引入构建、部署、数据库维护、备份恢复等复杂度。

当前方案更适合先快速跑起来，后续需求稳定后再拆分前后端或迁移数据库。

### 何时升级技术栈

- 用户超过 100 人，或文件/笔记数据明显增长：考虑迁移 PostgreSQL。
- 需要复杂交互和移动端适配：考虑 Vue/React 前端。
- 需要严格多人协作：引入 Yjs。
- 需要统一身份认证：接入 LDAP/OIDC。
- 需要公网访问：增加 HTTPS、反向代理、访问控制和安全审计。

## 6. 部署建议

### 局域网试运行

```bash
npm install
npm start
```

然后用服务器内网 IP 访问：

```text
http://服务器内网IP:3000
```

### 长期运行

推荐使用 PM2：

```bash
npm install -g pm2
pm2 start src/server.js --name lab-platform
pm2 save
```

### 备份重点

必须定期备份：

```text
data/
storage/uploads/
```

其中 `data/` 保存账号、目录、文件元数据、公告、笔记，`storage/uploads/` 保存真实上传文件。
